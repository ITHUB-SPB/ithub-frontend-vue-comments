# ithub-frontend-vue-comments

_(контрольная точка №3)_

## Задание

Приложение представляет собой дневник с записями (постами), а также комментариями к ним.

## Требования и критерии оценивания

### Стек

Использовать фреймворк `Vue 3` с `Composition API` и библиотеку `Pinia` для реализации хранилища.

### Компоненты

Определить и реализовать компоненты, например, это могут быть:

- Input (в двух вариациях: однострочным полем и текстовой областью)
- Button (в двух вариациях: базовой и опасной)
- Comment
- Post
- Modal (с формой на основе текстовой области)

Используя компоненты, реализовать интерфейс согласно [макету](https://www.figma.com/file/p4jDYcwlRsEVihOixcleWQ/Social-Book?type=design&node-id=0%3A1&mode=design&t=oi6vnFrdABIgP06f-1).

### Посты

Для каждого поста необходимо реализовать:

1. Добавление поста через форму (по нажатию на кнопку) - **один балл**

2. Удаление поста (по нажатию на кнопку) **один балл**

3. Изменение поста в модальном окне (по нажатию на кнопку сохранения, при наличии хотя бы одного изменения относительно исходного текста) - **до трёх баллов**. 

### Комментарии

1. Список привязанных комментариев. К каждому посту необходимо реализовать возможность отображения комментариев. По нажатию на кнопку мы можем либо отобразить, либо спрятать все комментарии. Данный функционал можно реализовать через модель (`isShow`) без выноса в хранилище. Кнопка является активной при наличии хотя бы одного комментария - **до двух баллов**.

2. Создание комментариев через форму - **один балл**.

3. Удаление комментариев - **один балл**

### Тема приложения

Реализовать функционал для темы приложения (темная / светлая, или что-то подобное) - **до двух баллов**.

### Хранилище

Создайте два глобальных хранилища: одно в Composition-стиле, второе в Options-стиле. 

Первое должно хранить состояние следующего вида

```js
[
    {
      id: 1,
      title: "Сегодня было замечательное предложение пойти поужинать этим вечером. Главное, чтобы погода была преимущественно теплой.",
      comments: [
        { id: 1, text: "Самый яркий комментарий в этом посте" },
        { id: 2, text: "Один из бессмысленный комментарий в этом посте"}
      ]
    },
    {
      id: 2,
      title: "Краткосрочное вымышленное преломление может выполнять особую роль в пространстве главной роли игрока ",
      comments: [
        { id: 1, text: "Очень научно и непонятно" },
      ]
    }
]
```

Второе - цветовую тему приложения.

Оба хранилища должны предоставлять необходимые геттеры и экшены.

## Формат сдачи

Прикрепить ответ ссылкой на собственную ветку (+ 0.5 балла) либо форк
